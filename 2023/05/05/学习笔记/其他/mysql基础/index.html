mysql事务

acid原则

a原子性:事务中的操作要么都发生，要么都不发生。

c 一致性:事务开始之前和事务结束以后，数据库的完整性约束没有被破坏。

i 隔离性:事务之间是隔离的，一个事务不应该影响其它事务运行效果。

  1. 脏读
  2. 不可重复读
  3. 幻读

d 持久性: 事务一旦提交，则持久化保存在数据库中。
